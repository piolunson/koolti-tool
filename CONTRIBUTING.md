# Contributing to KOOLTI-TOOL

Thanks for your interest in contributing! Before doing anything, please read this fully.

---

## ⚠️ Important — Read First

This project uses a **custom non-commercial license**.  
Unlike most open source projects, **you cannot just fork and send a Pull Request**.

**All contributions require prior written permission from the author.**

---

## How to Contribute

### Step 1 — Contact me first

Before writing a single line of code, send an email to:

📧 **piolunson@proton.me**

Include the following in your message:
- Your name / GitHub username
- What you want to add or fix
- Why you think it would improve the project

I'll get back to you as soon as I can.

### Step 2 — Wait for approval

If I approve your idea, I'll give you written permission to proceed.  
Without this approval, any modifications are not permitted under the license.

### Step 3 — Make your changes

Once approved, follow these guidelines:
- Keep the code style consistent with the existing codebase
- No unnecessary dependencies — stdlib and the existing ones only
- Test your changes before sending them
- Write clean, readable code — no comments needed, just clear naming

### Step 4 — Send your changes

After I approve and you're done, send me your changes via email or as a GitHub Pull Request — whichever we agreed on.

## Reporting Bugs

Found a bug? Great catch!  
Open a **GitHub Issue** with:
- What you were doing
- What you expected to happen
- What actually happened
- Your OS and Python version

No need to email for bug reports — Issues are fine for that.

---

## Suggesting Features

Have an idea for a new module or feature?  
Open a **GitHub Issue** with the `enhancement` label and describe your idea.  
If I like it, I'll reach out to discuss it further.
