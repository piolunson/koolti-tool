# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 8.x     | ✅ Yes     |
| 7.x     | ⚠️ Best effort |
| < 7.0   | ❌ No      |

## Reporting a Vulnerability

**Please do NOT open a public GitHub Issue for security vulnerabilities.**

If you find a security issue — a bug that could be exploited, a module that
causes unintended harm, or any other security concern — report it privately:

📧 **piolunson@proton.me**

### What to include in your report

- Description of the vulnerability
- Steps to reproduce it
- Which module or function is affected
- Potential impact
- Your suggested fix (optional but appreciated)

### What happens next

- I will acknowledge your email within **72 hours**
- I will investigate and fix the issue as quickly as possible
- You will be credited in the changelog if you wish
- A patched version will be released and the vulnerability disclosed publicly
  after the fix is available

## Scope

This tool is designed for **educational purposes and authorized testing only**.
Vulnerabilities that only apply when the tool is used illegally or against
systems without permission are out of scope.
